**If your idea changed / was TBD / still deciding, describe the current idea:**
> Action Board — paste messy meeting notes or a brain-dump; an Executa tool uses Anna's
> host LLM to extract structured action items (task, owner, deadline, priority) shown as
> reviewable cards on a To Do / In Progress / Done board. The human edits/approves/drags
> (human-in-the-loop); state persists in APS; one click posts a clean summary back to chat.

**What is the main user workflow of your app?\***
> **Input / trigger:** The user pastes raw meeting notes into the app window (or the
> assistant opens the board with `open_app_view(view="board", payload={notes})`).
>
> **Main steps:**
> 1. The UI calls the `action-triage` Executa tool, which borrows the host LLM via
>    reverse sampling (`sampling/createMessage`, no API key) to extract structured action
>    items — with a deterministic heuristic fallback so a board always appears.
> 2. Cards render on the board; the user reviews — edits owner/deadline, bumps priority,
>    approves, and drags across To Do / In Progress / Done (human-in-the-loop).
> 3. The board is saved to Anna Persistent Storage (survives reloads / devices).
> 4. "Send summary to chat" posts a structured artifact + readable message back into the
>    conversation.
>
> **Final output:** a reviewed, persisted action board, plus a chat summary of the items.
